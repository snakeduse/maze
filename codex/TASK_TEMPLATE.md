# TASK_TEMPLATE.md

Use this template when proposing a new task.

## Task Title

Short name of the task.

## Goal

What this task should achieve.

## Current Context

What already exists in the game.

## Scope

What must be changed.

## Out of Scope

What must not be changed.

## Expected Files

List files that will probably change.

## Acceptance Criteria

The task is done when:

- criterion 1
- criterion 2
- criterion 3

## Checks

Run:

    npm run build

Fix all build errors.

## Final Response

At the end, write:

- what was changed;
- what files were changed;
- what behavior was checked;
- what command was run.
